# 요약
1. 성능개선의 첫 단추는 처리량과 응답시간을 보는것이다.
2. 응답시간의 기준은 2가지 이다
   2.1. TTFB(Time To First Byte) : 첫번째 바이트를 받은시간을 기준
   2.2  TTLB(Time To Last Byte) : 마지막 바이트를 받은시간을 기준
3. 서버 성능을 올바르게 평가하려면 데이터 유형과 네트워크 환경에 따라 위 두 기준을 활용해야 한다.
4. 응답시간의 대부분의 경우 외부 API 연동, DB연동이 큰 비중을 차지한다.
5. 최대 TPS를 초과하면 지연시간이 생긴다.
6. 성능 개선을 하기 위해서는 TPS와 응답시간을 알아야한다.
7. TPS를 측정하는 툴을 사용하면 쉽게 분석 가능하다 (JMeter, Locust 등)
